# 마인크래프트 인게임 비디오 세팅
﻿
## 일반
﻿
**FOV (시야 범위)**
```
기본값: 70
범위: 30 ~ 110
Dynamic FOV(동적 FOV) 의 여부와 모니터의 크기에 따라 체감이 많이 달라지기에 개개인의 취향에 맞추어 사용하는 것이 좋습니다.
하지만 높을수록 더 많은 월드를 렌더링 하기 때문에 성능 저하를 일으킵니다.
보통의 유저의 경우 70에서 80 사이로 사용합니다.
```
**그래픽 (Graphics)**
```
기본값: Fancy(화려하게)
추천값: Fast(빠르게)
그래픽 세부 설정에서 값을 기본으로 사용하면 이 값으로 설정됩니다. 하지만 세부 설정에서 수동으로 값을 설정하면 의미는 없으니 크게 중요한 값은 아닙니다.
```
﻿**Render Distance (시야)**
```
추천값: 4 ~ 8
랜더 될 범위를 설정합니다.
오히려 컴퓨터가 좋을 경우 낮게 써도 문제가 없지만 컴퓨터가 좋지 않을 경우 움직일 때 랜더 한 시야 범위는 좁은데 다시 랜더 하는 시간이 느려 답답하실 수 있습니다.
하시는 게임과 자신의 체감에 맞추어 설정하시는 것이 좋습니다. 가능하면 최대한 낮추어 쓰는 것이 좋습니다.
```
﻿**Use VBOs (VBO 사용)**
```
추천값: On(켜짐)
정점 데이터를 GPU 메모리에 저장하고, 렌더링 작업을 수행할 때 이를 직접 참조합니다. 이는 렌더링 성능을 향상시키고, 프레임을 안정화하는 데 도움을 줍니다.
그러나 일부 구형 시스템에서는 VBO를 비활성화하는 것이 더 나은 성능을 제공할 수도 있습니다.
```

## Details (세부)
﻿
**Clouds (구름)**
```
기본값: Default(기본)
추천값: Off(꺼짐)
```
**Cloud Height (구름 높이)**
```
범위: 0(Off)% ~ 100%
추천값: Off(꺼짐)
구름의 높이를 설정하는 값이지만, 구름을 꺼두셨으면 의미 없는 값입니다.
```
**Trees (나무)**
```
기본값: Default(기본)
추천값: Fast(빠름)
이 값도 컴퓨터의 성능과 개개인의 취향에 따라 크게 갈리기에 본인이 원하는 값을 사용하면 된다.
Fast(빠름)을 사용 시 컴퓨터 성능을 가장 끌어올릴 수 있지만, 투명해야 할 나뭇잎이 사이가 검은색 텍스처로 변경된다.
기본(Normal)>멋지게(Smart)>화려하게(Fancy) 순으로 나뭇잎의 투명한 정도가 커집니다.
필자는 Fancy를 사용하고 있습니다.
```
**Rain & Snow (비와 눈)**
```
기본값: 기본(Default)
추천값: Off(꺼짐), 기본(Default)
꺼짐으로 하면 비와 눈이 눈에 보이지 않습니다. 성능보다는 자신이 편한 것을 쓰시는 걸 추천드립니다.
```
**Sky (하늘)**
```
기본값: 켜짐(On)
추천값: 꺼짐(Off)
상황에 따라 많이 달라지지만 일반적인 맵에서는 화면의 절반을 하늘이 차지합니다. 특히 리소스팩을 사용하는 경우, 예쁜 하늘을 위하여 훨씬 높은 비트로 고화질 하늘의 텍스처를 사용하는 것이 대부분입니다. 성능 측면에서는 매우 심하게 차이 나기 때문에 저사양 사용자라면 꺼짐을 권장합니다. 하지만 성능을 감수하고서라도 충분히 메리트가 있기에 취향껏 사용하시면 됩니다. 하늘은 월드의 윗부분만 렌더링 합니다. 아랫부분(공허)까지 렌더링 하고 싶으면 Star(별)를 키시고 Sky(하늘)은 끄셔도 하늘의 텍스처가 적용됩니다.
```
**Stars (별)**
```
기본값: On(켜짐)
추천값: Off(꺼짐), On(켜짐)
하늘과 공허의 텍스처 여부를 결정하는 설정입니다. 컴퓨터의 성능이 좋지 못하다면 반드시 꺼짐을 사용하시고 아니라면 취향껏 사용하시면 됩니다. 
만약 사용하신다면 Sky(하늘)은 끄셔도 됩니다.
```

## Quality (품질)

**Mipmap Levels (밉맵 레벨)**
```
범위: 1 ~ 4
추천값: Maximum (4)
텍스처의 여러 해상도를 미리 계산하여 거리에 따라 적절한 해상도를 사용하는 기능입니다.
그래서 멀리 있는 물체를 보다 부드럽게 보이게 하는 시각 효과입니다.
성능에 거의 영향을 주지 않으니 추천 값을 사용하시는 것을 권장하나, 진짜 심각한 똥컴이시라면 뭐라도 잡는 심정으로 Minimum (1)로 해보시는 것도 좋습니다.
```
**Mipmap Type (밉맵 유형)**
```
범위: 1 ~ 4
추천값: Nearest (1) (가장 가까움)
텍스처를 샘플링하는 품질을 조정하는 설정입니다.
높을수록 부드럽게 보이지만 성능을 떨어트려 추천 값을 사용하시는 것을 권장합니다.
```

## Performance (성능)

**Smooth FPS (부드러운 FPS)**
```
추천값: Off(꺼짐)
그래픽 드라이버 버퍼를 비워 요동치는 FPS를 안정화합니다.
하지만 이 기능을 사용하면 기본적으로 많은 FPS를 잡아먹기 때문에 끄는 게 절대적인 이득입니다.
```
**Smooth World (부드러운 세계)**
```
추천값: Off(꺼짐)
이 설정은 싱글 플레이에서만 적용됩니다.
내부 서버 부하를 분배해 FPS를 안정화 시키지만 큰 작업을 하는 게 아니라면 꺼짐을 사용하시는 걸 권장합니다.
```
**Fast Render (빠른 렌더)**
```
추천값: On(켜짐)
렌더링 알고리즘 최적화로 그래픽 카드의 사용량을 줄여 FPS를 크게 향상시킬 수 있습니다.
하지만 HUDCaching 모드를 사용하기 위해서는 이 모드를 끄셔야 정상적으로 작동합니다.
HUDCaching은 루나 클라이언트에 내장되어 있으며, 포지나 페더클라이언트에서는 Patcher라는 모드를 통해 사용하실 수 있습니다.
이 모드는 HUD 요소를 계속해서 그리지 않고, 미리 캐시에서 불러오므로, 렌더링 성능이 크게 향상됩니다.
```
**Fast Math (빠른 계산)**
```
추천값: Off(꺼짐)
최적화된 sin()과 cos() 함수를 사용하여 CPU 캐시를 더 잘 활용하고 FPS를 높일 수 있다고 말하지만, 성능에 큰 차이가 없으며 이득의 비해 훨씬 많은 오류들을 초래합니다.
```
**Chunk Updates (청크 갱신)**
```
추천값: 1
자신의 FPS 만큼 갱신할 청크의 양을 결정하는 설정입니다.
특별한 이유가 없으면 1로 사용하시는 것을 권장합니다.
```
**Dynamic Updates (동적 갱신)**
```
추천값: Off(꺼짐)
사용자가 가만히 서 있을 때만 더 많은 청크 갱신을 합니다.
```
**Render Regions (Render Regions)**
```
추천값: On(켜짐)
월드를 작은 영역으로 나누어 각각의 영역을 독립적으로 렌더링합니다. 이를 통해, 화면에 표시되는 각 영역의 렌더링 작업이 병렬로 처리되거나 필요에 따라 갱신됩니다.
이 기능은 특히 넓은 영역을 한 번에 렌더링하는 대신, 작은 영역 단위로 렌더링을 처리하므로 CPU와 GPU의 부담을 줄입니다. 결과적으로, 게임의 성능이 향상되고, 렉이나 지연 현상이 줄어듭니다.
또한 메모리 사용량을 최적화하여, 렌더링 중 필요하지 않은 영역의 데이터를 메모리에서 제거하거나 덜 자주 업데이트하도록 합니다. 이는 특히 큰 월드나 복잡한 구조를 렌더링할 때 유용합니다.
```
**Lazy Chunk Updates (느긋하게 청크 불러오기)**
```
추천값: Off(꺼짐)
청크 업데이트를 지연시켜 CPU 사용량을 줄이고 게임의 프레임률을 안정화시키는 데 도움을 줍니다.
```
